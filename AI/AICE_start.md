## Python Basic


**변수**

Variable. 변하는 수치. <->상수

하나의 방(주소)안에 우리가 기억해야 할 것을 저장.

> A = 100 #A라는 방에 100이라는 값(데이터)저장.

변수 할당 : 변수에 값을 지정. “=”을 사용하여 지정.

* 변수명 B1과 문자열 ‘B1’의 구분에 주의


**리스트(list)**

하나의 변수에 여러 개 값을 저장, 접근, 수정

선언

>    num = [1, 2, 3, 4, 5] --> 대괄호 이용

    num = list(1, 2, 3, 4, 5)

    num = list(range(1, 11))

    num = list(range(1, 11, 2)) * range(시작값, 종료값, 간격) 간격은 디폴트1(생략가능)

리스트 함수(listname.TAB으로 확인)를 사용해 원소에 접근, 수정, 삭제 가능.

주요 리스트 함수

> .append(요소) : 리스트 끝에 요소 추가

    .insert(인덱스, 요소) : 리스트의 특정 인덱스에 요소 추가 *그러면 그 인덱스 값과 이후값들은 하나씩 뒤로 밀림

인덱스 : 리스트의 각 요소 위치에 접근. 첫번째 요소의 인덱스는 0. 또한 음수(-)표현을 통해 역순 접근도 가능하다.

> Num[0] 으로 첫번째 값에 접근

    Num[-1] 으로 마지막 값에 접근

리스트 내 원소의 자료형은 제한이 없다.(또한 통일될 필요도 없다.)


**파이썬 딕셔너리(Dictionary)**

하나의 변수에 여러 값을 저장, 접근, 수정, 삭제

Key – Value 구성. -> 값이 많아도 key 만 알면 바로 접속 가능

선언

> DIC = { k1 : v1, k2 : v2 }

    DIC2 = dict()

    DIC2[k] = v

    > dict = {‘name’: ‘pey’, ‘phone’: ‘010-1234-1234’, ‘age’: 25 }

        dict[‘name’]

        출력 : ‘pey’

* 키의 값은 가변해서는 안된다. -> 리스트, 변수를 키로 가질 수 없음. 즉, 튜플, 문자열, 숫자 등을 키로 사용.

* list와 마찬가지로 한 딕셔너리 내에 여러 자료형 원소 저장 가능.

딕셔너리 관련 함수

> .keys() : 딕셔너리 key만 모아 반환

    .values() : value 만 모아 반환

    .items() : key, value 쌍을 튜플로 묶은 값 반환.

    그 외 clear, get, in 등

**반복문**


for문 : 반복적으로 문장을 수행한다

문법

> for 변수 in 리스트(또는 문자열):

        수행할 문장 1
        수행할 문장 2

> for i in [1, 2, 3, 4, 5]:

    print(‘숫자를 {}을 카운트한다’.format(i))

> 익숙한 사용법은 for i in range(1,6):

파이썬의 range(시작값, 종료값) 함수를 이용해 연속된 정수 생성

단, 종료값을 포함하지 않음.(1~10 필요하면 1, 11로 입력해야)


## Python 자료형 및 활용

**인덱싱**

Eng) indexing : 무언가를 ‘가리킨다’. -> 특정한 값을 뽑아내는 역할

여러 개의 값이 모여 집합을 이룰 경우, 집합 내 각 값의 위치를 알 수 있고 찾아갈 수 있게 한다.

형태 : x[번호]

* 파이썬은 숫자를 0부터 셈. 제일 앞의 것이 0, 이후로 +1씩.

* 역순부터 하고자 한다면 -1, 이후로 -2, -3….(단, -0은 0과 같다.)


**슬라이싱**

Eng) slicing : 무언가를 ‘잘라낸다’ -> 한 문자를 넘어 단어를 뽑아내는 방법

형태 : x[시작번호:끝번호]

* 단, 끝번호는 제외된다.

> x = Rome is not built in a day!
    
    x[0:4] -> ‘Rom’

    x[18:-3] ->’in a d’ *-3은 ‘a’이니 그 전까지.

리스트, 문자열, 튜플, 기타 등등에 대하여 사용이 가능.

*리스트 자료형과 관련 함수 등은 기초 파트 리스트 항목에 통합하여 작성.


**튜플**

리스트와 유사하지만 요소의 수정, 추가 불가.

소괄호(( ))이용

> a = (1, 2, 3)

    b = 1,2,3 *이렇게도 가능*

장점 : 	

> **반복 처리 시** 리스트보다 근소하게 **빠름**(연산량 많을 경우 성능 향상)

    요소 변경 불가 - **딕셔너리의 키로 사용** 가능

    보안이 필요한 데이터에 사용됨



*딕셔너리 자료형과 그 관련 함수 등은 기초 파트 딕셔너리 항목에 통합하여 작성
