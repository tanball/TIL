# 관계(Relationship)

## 정의

**관계** : 두 개 이상의 엔터티 간에 논리적으로 연결된 의미 있는 연관성.

관계를 통해 서로 다른 데이터를 연결해 새로운 정보 도출.

현재 실무에서 쓰이는 대부분의 정형 데이터베이스는 *관계형 데이터베이스*.


*정형(Formal) 데이터베이스 : 정해진 형식과 구조를 가지는 데이터베이스*

*관계형(Relational) 데이터베이스 : 정형 데이터베이스의 일종으로, 엔터티 간 관계를 이용해 데이터를 조합해 새로운 정보를 만드는 데이터베이스*


## 관계 분류

### 목적에 따른 분류
1. **존재적 관계** : 단순히 소속되거나 포함되는 관계(학생-학과(소속되다), 직원-부서(소속되다)
- =연관 관계
- ERD(데이터 모델링)에서는 **실선**으로 연결
- UML(프로세스 모델링)에서는 **실선 화살표**로 연결
2. **행위적 관계** : 행위로 인해 연결되는 관계(회원-상품(구매하다), 사용자-게시글(작성하다))
- =의존 관계
- ERD(데이터 모델링)에서는 **실선**으로 연결
- UML(프로세스 모델링)에서는 **점선 화살표**로 연결

## 관계 표기 및 읽는법

### 관계 표현 3요소
1. 관계명(Membership) : 두 엔터티 간의 논리적인 관계를 동사로 명시
2. 관계차수(Degree/Cardinality) : 인스턴스가 얼마나 관계에 참여하는지(1:1, 1:N, N:M) 명시
3. 관계 선택성(Optionality) : 인스턴스가 반드시 관계에 참여해야 하는지 여부 명시

### 관계 표기법

IE 표기법

Barker 표기법


**관계명**
1. 관계명은 엔터티끼리의 논리적인 관계를 **현재형 동사**로 표현한다. (소속하다, 구매하다, 보유하다 등)
2. 관계명은 애매한 표현이나 과거/미래 시제로 작성하지 않도록 주의해야 한다.
3. 관계명은 두 엔티티 간의 연결이므로 양방향 해석이 가능하다. (소속하다-소속되다 등)

*IE, Barker 동일*


**관계차수**
1. 관계차수는 엔터티 내의 인스턴스가 각각 관계에 '얼마나' 참여했는가를 물어본다.
2. 관계차수의 종류는 1:M(일대다), 1:1(일대일), N:M(다대다) 관계가 존재한다
3. IE표기법, Barker 표기법 모두 다(N, M)쪽 관계 차수는 까치발(<)로 표시한다.

> IE 표기법
>
> '하나'를 문자 'I'로 표기 (까치발 꼭짓점을 지나는 실선의 수직선)
>
>> 하나의 A는 **최소 하나**, 혹은 그 이상의 B와 X의 관계를 맺는다. 
>>
>> 이 경우 B 방향에 I, <, 실선 조합

> Barker 표기법
>
> 일(하나)을 따로 표현하지 않는다.


**관계 선택성**
1. 관계 선택성은 엔터티 내의 인스턴스가 관계에 **반드시** 참여해야 하는가를 물어본다.
2. 반드시 참여한다 = **필수** 참여 | 반드시 참여할 필요 없다 = **선택** 참여

> IE 표기법
>
> 선택 관계는 O(동그라미) 기호
>
>> 하나의 A는 하나의 B를 보유하거나, 여러 B를 보유할 수 있다.
>>
>> 혹은 하나의 B도 보유하지 않을 수 있다.
>>
>> 이 때, A는 I(하나의 A)를 갖고, B는 <(여러 B)와 I(하나의 B), O(혹은 하나의 B도 ...)를 갖는다.
>>
>> 즉, B(선택참여)에 O를 표시한다.

> Barker 표기법
>
> 반대쪽 선을 점선으로 표기
>
>> 하나의 A는 하나의 B를 보유하거나, 여러 B를 보유할 수 있다.
>>
>> 혹은 하나의 B도 보유하지 않을 수 있다.
>> 
>> 이 때, A 쪽에 점선, B쪽에 <와 실선.
>>
>> 즉, 선택참여의 반대쪽을 ----(점선)으로 표시한다.

## 관계 체크 사항
1. 두 개의 엔터티 사이에 관심 있는 연관 규칙이 있는가?
- 두 엔터티 간에 업무에 필요한 논리적인 규칙이 존재하는지
2. 두 개의 엔터티 사이에 정보의 조합이 발생하는가?
- ex) [회원] + [상품] -> "구매하다"
3. 업무기술서, 장표에 관계연결에 대한 규칙이 서술되어 있는가?
- '규칙'은 관계차수 및 선택성이 적절히 표현되어 있는지를 의미
4. 업무기술서, 장표에 관계연결을 가능케 하는 동사(Verb)가 있는가?
- '동사'는 관계명이 적절하게 표현되는지를 의미

## ERD

**ERD** : Entity Relationship Diagram. 엔티티 간의 논리적인 관계를 그림으로 도식화한 것.

다양한 표기법이 존재하나, 시험에서는 IE, Barker 사용.

### ERD의 필요성

1. 복잡한 요구사항을 시각화
2. 구성원 간 의사소통 도구
3. 개념적 모델링의 산출물


### ERD 그리는 순서
2. 엔터티 도출
2. 엔터티 배치
3. 관계선 그리기
4. 관계명 작성
5. 관계차수 작성
6. 관계 선택성 표시

*시험레서 이 그리는 순서를 문제로 물어볼 수 있음.*


## 엔터티와 관계
엔터티는 일반적으로 다른 엔터티와 하나 이상의 관계를 멪어야 한다.

예외 : 코드성 엔터티, 통계성 엔터티

### 코드성 엔터티

국가코드 엔터티처럼 대상을 코드화하여 정리

여러 엔터티에서 공통으로 참조되며, 국가 정보를 일관되게 관리하기 위한 기준 데이터 역할을 한다.

**여러 엔터티에 연계될 수 있기에 오히려 관계를 표현하면 데이터 모델이 복잡해진다.**


### 통계성 엔터티

특정 기간의 수치 데이터를 저장하는 용도로 사용됨.

**분석 및 리포트 목적으로 활용되기에 다른 엔터티와의 관계 없이도 독립적인 의미를 지닌다.**
