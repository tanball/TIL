# 식별자

## 정의

**식별자** : 엔터티 안의 인스턴스를 유일하게 식별할 수 있는 속성의 집합

## 식별자 분류

### 대표성 여부
1. **주식별자** : 여러 보조식별자 중 대표로 선택된 단 하나의 식별자
2. **보조식별자** : 주식별자가 될 수 있지만 선택되지 않은 나머지지

### 자생 여부
스스로 생성되었는가?

1. **내부식별자** : 엔터티 내부에 원래부터 존재한 식별자
2. **외부식별자** : 다른 엔터티와의 관계를 표현하기 위해 빌려온 **다른 엔터티의 식별자**

### 단일 속성 여부

1. **단일식별자** : 속성 1개로 구성
2. **복합식별자** : 2개 이상의 속성으로 구성

### 대체 여부

1. **본질식별자** : 원래 존재하선 속성으로 구성됨. **업무적 의미가 있음**
2. **인조식별자** : 새로 만든 임의의 식별자. **업무적인 의미가 없음**


*반드시 하나의 가준으로만 식별자 분류를 수행할 필요는 없다*


## 주식별자

엔터티의 인스턴스를 유일하게 구별하는 **엔터티의 대표 식별자**

엔터티는 반드시 하나의 주식별자를 가져야 한다.

### 주식별자의 특징

1. 유일성 : 엔티티 내 각각의 인스턴스를 유일하게 식별할 것
2. 최소성 : 최소한의 속성 조합으로 식별자를 구성할 것
3. 불변성 : 주식별자로 지정된 속성은 가급적 변하지 않을 것(변할 수는 있으나, 실무에서 변할 일 없고, 시험에서도 불변한다로 요구.)
4. 존재성 : 주식별자로 지정된 속성은 반드시 값이 입력될 것(Null 불가)

### 주식별자 도출 기준

1. 업무에서 자주 사용될 것 (조회, 검색 등에 자주 쓰여야)
2. 이름, 내역처럼 불명확한 속성은 배제할 것 (중복 쉽고 오탈자 가능성 큰 것들 빼기)
3. 너무 많은 속성으로 이루어지지 말 것 (=최소성을 만족)


## 식별 관계 VS 비식별 관계

**식별 관계** : 외부식별자가 주식별자로 포함되어 사용됨 (=**강한연결관계**)

> [회원]엔터티의 '회원ID'속성(주식별자)를 [회원연락처]의 외부식별자로 사용. 
>
> 이 때, [회원연락처]는 '회원ID', '구분코드' 집합을 주식별자로 한다.
>
> 즉, '회원ID'는 [회원연락처]의 FK이면서, PK의 일부임.
>
> 회원 엔티티와 회원연락처 엔티티는 강한연결관계에 있다. 식별 관계이다.

**비식별 관계** : 외부식별자가 일반속성으로 사용됨 (=**약한연결관계**)

> [회원]의 '회원ID'는 [문의]의 외부식별자.
>
> [문의]의 주식별자는 '문의번호'
>
> 두 엔터티는 비식별 관계. 약한 연결 관계에 있다.

*[외부식별자는 외래키 개념으로 확장됨.](속성.md#엔터티_구성_방식)*

### 식별 관계와 비식별 관계의 표기 방식

> IE에서의 표기
>
> 강한연결관계(식별 관계)를 실선으로 표시한다.
>
> 약한연결관계(비식별 관계)를 점선으로 표시한다.

> Barker에서의 표기
>
> 강한연결관계(식별 관계)를 수직바로 표시한다.
>
> 약한연결관계(비식별 관계)를 별도로 표시하지 않는다.

### 식별 관계와 비식별 관계의 특징

| 항목 | **식별 관계** | **비식별 관계** |
|------|------|------|
| **외부식별자의 위치** | 주식별자에 포함 | 일반속성 | 
| **값 필수 입력 여부** | 필수 | 입력 선택 가능(null허용 가능) | 
| **데이터 독립 생성 기능 여부** | 불가능(부모 없이 생성 불가) | 가능(부모 없이도 생성 가능) |
| **생명주기 연계** | 부모 삭제시 자식도 삭제 | 부모 삭제되도 자식은 남길 수 있음 |
| **식별자 확장** | 상속 구조로 주식별자 길이 증가(=확장) | 주식별자 확장 안됨(=차단) |

> 식별자 상속 : 부모 엔터티의 주식별자가 자식 엔터티의 식별자에 포함되는 것
>
> 식별자 차단 : 부모 식별자를 참조하되 자식 식별자에는 포함되지 않는 것
>
>
> 식별자 확장 : 식별 관계는 부모 엔터티의 식별자를 자식 엔터티에게 빌려줄수록 자식 엔터티의 주식별자 속성 조합이 증가하게 된다.
> 이를 '부모 엔터티의 식별자를 자식 엔터티에 이전한다'고 한다.
>
> 비식별 관계는 주식별자로부터 독립적이라 상속이 불필요하기에 '부모 엔터티의 식별자를 자식 엔터티에 차단한다'고 함.

[*생명주기 연계를 구현하려면 외래키에 ON DELETE CASECADE 제약조건을 설정하면 됨.*](foreign key 참고)


### 식별 / 비식별 사용 선택

**식별 관계에 적합한 경우**
1. 데이터 무결성 중시
2. 삭제 연쇄 필요
3. 조인 줄이기 원함

**비식별 관계에 적합한 경우**
1. 독립적 생성 필요
2. 식별자 간결함 중시


식별 관계만 사용하면 : 데이터 탐색 위한 주식별자 조건이 많아짐.

비식별 관계만 사용하면 : 조인이 증가함.
