# 엔터티


## 정의

**엔터티(Entity)** : 업무에 필요한 정보를 저장 및 관리하기 위한 집합적/포괄적인 대상

**인스턴스(Instance)** : 엔터티에 속한 개별적인 데이터

간단히 말하자면, DB의 테이블이 되는 것(회원)은 엔터티. 그 레코드 하나하나가 각각의 인스턴스. 

ex)'회원ID' 속성값이 AAAAA인 인스턴스. '이름', '성별', '나이'를 갖는다.


## 엔터티 특징

1. 업무에 필요로 할 것
2. 2개 이상의 인스턴스를 가질 것
3. 속성을 1개 이상 가질 것
4. 다른 엔터티와 한 개 이상 관계를 가질 것 *통계성/코드성 엔터티는 관계를 갖지 않아도 예외로 인정됨.*
5. 식별자가 존재할 것.

*관계 : Section 4 참고.*



## 엔터티 분류

### 유무형
1. 유형 엔터티 : 눈에 보이는 **실체**(학생, 교재, 교수 등)
2. 개념 엔터티 : 눈에 보이지 않고 **개념적**으로 존재(강의, 학과 등)
3. 사건 엔터티 : 일종의 **무형** 엔터티. **업무 수행 중 발생**. 발생량 많음(구매, 주문, 수강 등)

### 발생 시점
1. 기본(키) 엔터티 : **프로젝트 초반**에 정의. **독립 생성** 가능. **주식별자** 보유(회원, 상품 등)
2. 메인(중심) 엔터티 : **기본 엔터티로부터 파생**. 업무의 중심적 역할(구매, 주문 등)
3. 행위 엔터티 : 엔터티 간 **행위 발생 시 생성**. 시간에 따라 누적됨 (주문이력, 배송상태이력 등)


## 엔터티 명명 규칙
1. 현업 용어 사용(이용자=현업에게 익숙한 용어로)
2. 약어 가급적 지양(단, 조직 내 합의된 약어는 가능))
3. 단수형 명사 사용
4. 이름은 유일하게
5. 생성 의도에 맞는 이름 부여
