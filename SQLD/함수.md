# 함수(Function)

## 개념

**함수(Function)** : 입력값을 받아 특정 연산을 수행하고 하나의 결과값을 반환하는 구조.

## 함수의 종류(in SQL)

**내장형 함수(Built-in Function)** : DBMS 제조사에서 미리 만들어 제공. 함수명, 입력값, 기능을 알면 즉시 사용 가능.
사용자정의 함수(User-defined Function) : 사용자가 직접 정의하여 사용(*SQLD에선 별도로 다루지 않음*)

## 내장형 함수

### 문자형 함수
문자 데이터를 처리하기 위한 함수

*오라클과 SQL server가 다르면 왼쪽이 오라클*

1. UPPER(문자열)
- 영어 소문자를 대문자로 변환
```
SELECT UPPER('hello123abc') AS RESULT
FROM DUAL;
```
- RESULT : HELLO123ABC 

2. LOWER(문자열))
- 영어 대문자를 소문자로 변환

3. CONCAT(문자열1, 문자열2)
- 문자열 2개 연결

4. SUBSTR(문자열, 시작위치, 개수)
- 문자열 일부 추출
```
SUBSTR('19940216',1,4)
```
- 출력결과 : 1994
- 시작 위치는 문자열의 맨 앞 글자가 1, 이후로 1씩 증가.

5. ASCII(문자)
- 문자의 아스키코드 반환
```
ASCII('A')
```
- 출력결과 : 65

6. CHR(숫자) / CHAR(숫자)
- 아스키코드를 문자로 반환
```
CHR(65)
```
- 출력결과 : 'A'

7. LENGTH(문자열) / LEN(문자열)
- 문자열의 길이 반환

8. LTRIM(문자열, 지정문자) / LTRIM(문자열)
- 왼쪽 공백 혹은 지정 문자 제거
- SQL server는 왼쪽 공백 제거만 가능
```
LTRIM(' my code_')
LTRIM(' my code_', '_')
```
- 출력결과 : 'my code_', ' my code'

9. RTRIM(문자열, 지정문자) / RTRIM(문자열)
- 우측 공백 혹은 지정 문자 제거
- 기본적으로 8과 같다.

10. TRIM([leading | trailing | both] 지정문자 FROM 문자열)
- 앞/뒤/양쪽의 특정 문자 제거
```
TRIM(LEADING '#' FROM '#A#B#')
TRIM(TRAILING '#' FROM '#A#B#')
TRIM(BOTH '#' FROM '#A#B#')
```
- 출력결과 : 'A#B#', '#A#B', 'A#B'
- 첫 번째 속성으로 앞을 지울지, 뒤를 지울지, 양쪽을 전부 지울지 결정.
- 내부에 FROM이 들어간다는 특징.

11. REPLACE(문자열, 기존문자, 변경문자)
- 문자열에서 특정 문자를 치환
```
REPLACE('I am studing Oracle.', 'Oracle', 'SQL')
```
- 출력결과 : I am studing SQL.

*SQL은 함수명 등은 대소문자를 구분하지 않으나, 따옴표('')안의 문자열은 대소문자를 구분한다.*

### 숫자형 함수

1. ABS(숫자)
- 입력한 숫자의 절댓값 반환

2. SIGN(숫자)
- 입력한 숫자의 양수/음수/0 여부 구분(1/-1/0 반환)

3. MOD(숫자1, 숫자2)
- 숫자1을 숫자2로 나눈 나머지 반환

4. CEIL(숫자)
- 입력한 숫자의 소숫점 이하 값을 올림 처리

5. FLOOR(숫자)
- 입력한 숫자의 소숫점 이하를 버린다(내림 처리)

6. ROUND(숫자, 자릿수)
- 입력한 숫자를 지정한 자릿수까지 반올림
- 자릿수로 음수 지정 시 소숫점 기준 왼쪽으로 자릿수만큼 이동.
```
SELECT ROUND(123.456, 2) AS RESULT1
     , ROUND(123.456, -2) AS RESULT2
  FROM DUAL;
```
- RESULT1 : 123.46 | RESULT2 : 100
- 자릿수 2 -> 소숫점 아래 2자리수까지 남기고, 3자리수부터 반올림함.
- 자릿수 -2 -> 소숫점에서 왼쪽으로 2자리이니 100의 자릿수까지 남기고 그 아래 반올림.

7. TRUNC(숫자, 자릿수)
- 자릿수까지만 값을 유지하고, 그 이후는 모두 버림
- 자릿수의 입력은 ROUND와 동일한 방식.

8. POWER(밑, 지수)
- 밑수를 지수만큼 제곱한 값을 반환합니다.
- ex) power(2, 4) -> 2의 4승(16)반환.

### 날짜형 함수
Oracle과 SQL Server간 함수 차이가 크다.

또한 Oracle에서는 날짜에 숫자를 더하면 '일'단위로 계산됨.
즉 +1 은 하루 추가, +1/24는 1시간 추가. +1/24/60은 1분 추가.

1. SYSDATE / GETDATE()
- 현재의 시스템 날짜와 시간을 반환(연-월-일 시:분:초)
- SYSDATE는 뒤에 (괄호)를 붙이지 않는다.

2. EXTRACT(단위 FROM 날짜) / DATEPART(단위, 날짜)
- 날짜에서 연도, 월, 일을 추출
- 단위에는 YEAR, MONTH, DAY 등을 입력 가능.
- 날짜에는 날짜형 변수 외에도 sysdate/getdate()등을 사용할 수도.

### 형변환 함수
자료형을 변환하는 기능

멍시적 형변환과 암시적 형변환으로 나뉜다.

명시적 형변환 : 아래 정리된 함수를 이용하여 변환

암시적 형변환 : 산술 연산이나 비교 연산 등으로 자료형을 변환.(1+'5'를 하면 DBMS가 '5'를 5로 자동 변환하여 계산함)


1. TO_CHAR(날짜, 포맷)
- 날짜 데이터를 문자열로 변환
- 기본 날짜 데이터 구조 : 2026-05-08 15:33:15
- 기본 날짜 데이터의 전체 포맷 : YYYY-MM-DD HH24:MI:SS

**포맷(Format) 종류**
- YYYY : 연도 4자리(0001~9999)
- MM : 월 2자리(01~12)
- DD :  일 2자리(01~31)
- HH24 : 시간(0~24)
- MI : 분(0~59)
- SS : 초(0~59)

2. TO_DATE(문자, 포맷)
- 지정 포맷과 동일한 형태의 문자를 입력받아 날짜형으로 변환해준다.
- 입력받지 않은 부분은 0 처리.
- 포맷과 문자열이 반드시 일치해야 함.
```
SELECT TO_DATE('20250302', 'YYYYMMDD) AS DATE1
     , TO_DATE('202503021130' 'YYYYMMDDHH24MI') AS DATE2
  FROM DUAL;
```
- DATE1 : 2025-03-02 00:00:00 | DATE2 : 2025-03-02 11:30:00

3. TO_NUMBER(문자)
- 문자를 숫자로 변환
- 단, 입력한 문자열에는 숫자값만 포함되어야 함('1234'는 되지만 '12가34'는 불가.)

4. CAST(값 AS 자료형) **SQL Server한정**
- 값을 지정한 자료형으로 변환한다.
- SQL 서버는 위의 TO계열의 함수들 대신 CAST로 변환.
- VARCHAR, DATETIME, etc...

### NULL함수
NULL값으로 인해 산술/비교 연산에서 오류가 나거나 NULL값이 반환되는 걸 보완하는 함수

1. NVL(값1, 값2) / ISNULL(값1, 값2)
- 값1이 null이면 값2를 반환.

2. NULLIF(값1, 값2)
- 값1과 값2가 같으면 null, 다르면 값1을 반환

3. COALESCE(값1, 값2, ...)
- 여러 값 중 첫 번째로 null이 아닌 값을 반환.(null아닌 게 없으면 null 출력.)

## CASE와 DECODE 문법
조건에 따라 서로 다른 결과를 반환할 수 있는 구문들.

시험, 실무에서 모두 자주 사용됨.

### CASE 문법
Oracle, SQL Server 공통 문법

- CASE로 시작하고 END로 끝남.
- 내부에 WHEN ~ THEN ~ ELSE로 조건과 값 작성.
- ELSE를 따로 지정하지 않을 경우, 조건에 맞는 값 없으면 NULL반환.
- **SIMPLE CASE** 와 **SEARCHED CASE**가 있다.

1. SIMPLE CASE문법
case뒤에 입력된 기준값이 when 뒤의 값과 일치하면 then값을 출력. when에 해당하는 게 없다면 else값을 반환.
```
SELECT CASE 기준값 WHEN 조건1 THEN 결과1
                 WHEN 조건2 THEN 결과2
                 ELSE 기본값(미입력시 NULL)
       END AS 결과
  FROM DUAL;
```
실제 값을 대입해보면
```
SELECT CASE '재학생' WHEN '재학생' THEN '재학중입니다.'
                   WHEN '졸업생' THEN '졸업생입니다.'
                   ELSE '정보없음'
       END AS RESULT
  FROM DUAL;
```
- RESULT : '재학중입니다.'

2. SEARCHED CASE 문법
case when 뒤에 조건식(search) 지정 가능. 조건식이 참이면 해당 then을 반환. 맞는 조건이 없다면 else값 반환.
```
SELECT CASE WHEN 조건1 THEN 결과1
            WHEN 조건2 THEN 결과2
            ELSE 기본값(미입력시 NULL)
       END AS 결과
  FROM 테이블명;
```
실제 값을 대입해보면
```
SELECT 학생ID, 점수
        CASE WHEN 점수 >= 90 THEN 'A'
             WHEN 점수 >= 80 THEN 'B'
             WHEN 점수 >= 70 THEN 'C'
             ELSE F
       END AS 등급
  FROM 학생;
```
select, where, order by 등 다양한 위치에서 사용 가능.

*복습 필요 구간*

### DECODE 문법(Oracle)
조건에 따라 다른 값 반환.

```
DECODE(기준값, 조건1, 결과1, 조건2, 결과2, ... , 디폴트값)
```
예시
```
SELECT DECODE('안녕', '안녕', '나도안녕', '누구세요') AS TEST1,
       DECODE('4', '0','영','1','일','2','이','3이상') AS TEST2
FROM DUAL;
```
- TEST1 : 나도안녕 | TEST2 : 3이상


### 비교
| 비교 항목 | DECODE | CASE |
|--------|--------|------|
| 지원DB | oracle 전용 | oracle, sql server 모두 지원 |
| 비교 조건 | =(동등 조건)만 가능 | simple case로 =조건을, searched case로 부등호 조건, between등도 가능 |
| 기본값 처리 | 마지막 입력값 | ELSE키워드 사용(미입력시 null) |
| 유연성 | 낮음 | 높음 |

따라서, simple case문법은 decode와 서로 완전히 대체가 가능.
단, searched case 조건은 = 외의 조건도 가능하기에 100% 호환은 불가.
