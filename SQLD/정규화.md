# 정규화

## 개념

1. 정규화 : 엔터티를 분리해 데이터의 품질을 향상시키는 작업. 데이터의 이상 현상을 방지하기 위해 논리적 데이터 모델링 과정에서 반드시 수행되는 과정이다.
2. 이상현상 : 중복된 데이터나 부정확한 구조로 인해 예기티 못하게 데이터의 일관성이 깨지는 문제

### 이상현상

1.삽입 이상
- 데이터 입력 시 원치 앖은 정보가 입력되거나, 원하는 정보를 입력할 수 없는 현상.

2. 갱신 이상
- 데이터가 변경되면서 데이터의 일관성이 깨지는 것.

3. 삭제 이상
- 데이터를 삭제할 때 원하지 않은 데이터도 삭제되는 현상.


## 정규화 진행

### 함수적 종속이란

**f(x) = y, x에 의해 y가 결정되며 이때 y는 x에 함수적으로 종속한다고 한다.**

### 정규화 과정
비정규형

 | 도메인 원자성 확보

1NF(1차 정규형)

 | 부분종속성 제거

2NF(2차 정규형)

 | 이행종속성 제거

3NF(3차 정규형)

 | 결정자 모두가 후보키여야 함(=보이스코드 정규화)

BCNF(Boyce-Codd NF)

 | 다치 종속 제거

4NF(4차 정규형)

 | 조인 종속 제거

5NF(5차 정규형)


**1. 1차 정규화**
- 비정규형에서 **도메인 원자성**을 확보해 1차 정규형으로 전환하는 과정
- **도메인 원자성** : 도메인이 다중값이거나 비슷한 속성이 여러 개라면 원자적으로 하나만 두라는 것

> 학생ID, 이름, 비상연락처의 속성을 가진 가상의 엔터티를 대상으로 예시 설명
>
> 비상연락처가 둘 이상일 경우, 다중값 표기(속성 내 콤마 구분)하지 말고 별도의 레코드로 구분하기
>
> 속성이 비상연락처1, 비상연락처2, 비상연락처3 ... 일 경우 이것도 비상연락처 하나의 속성으로, 여러 개 레코드 두기.

**2. 2차 정규화**
- 1NF에서 **부분종속제거**를 통해 2차 정규형으로 전환하는 과정
- **부분종속** : 식별자를 제외한 나머지 모든 속성들이 식별자에 의해 완전히 종속되지 않고, 식별자의 부분 속성만으로도 일부 속성을 종속하는 것

> | 학번 | 과목코드 | 학생이름 | 과목명 |
> |------|------|------|------|
> | 1001 | CS101 | 홍길동 | 자료구조 |
> | 1002 | CS101 | 김철수 | 자료구조 |
> | 1001 | MA101 | 홍길동 | 수학개론 |
>
>> X(학번, 과목코드) -> Y(학생이름, 과목명)
>
> 주식별자(학번, 과목코드)로 나머지 일반속성들을 완전함수종속.
>
>> X(과목코드) -> Y(과목명)
>
> 그런데 위와 같이 주식별자의 부분속성(과목코드)만드로 일부 일반속성(과목명)을 식별 가능. 즉, 부분함수종속


방법

1) 부분 종속에 참여한 속성(배신자 속성)들을 따로 분리

> 과목코드, 과목명을 따로 분리.(분리된 [과목] 엔터티와 기존의 [학생]엔터티)

2) 데이터 연관성 유지를 위해 기존 엔터티에 분리된 식별자를 다시 붙인다.

> 과목 엔터티(과목코드PK, 파일명), 학생 엔터티(학번, 학생이름, 과목코드)


**3. 3차 정규화**
- 2NF에서 **이행종속제거**를 통해 3차 정규형으로 전환하는 과정
- **이행종속성** : 식별자가 아닌 일반속성들끼리 함수 종속이 발생하는 경우

> 이행 종속 공식
>
>> X(식별자) -> Y(일반속성)이고 Y(일반속성) -> Z(일반속성) 이면 X->Z이다.

방법

1) 이행 종속에 참여한 속성(배신자 속성) 분리

> 사원 엔터티(사번pk, 부서코드, 부서명)
>
> 사번->부서코드->부서명 종속
>
> 사원 엔터티(사번pk), 부서 엔터티(부서코드pk, 부서명)

2) 데이터의 논리적 연결을 위해 외부식별자로 연결

> 사원 엔터티(사번pk, 부서코드fk), 부서 엔터티(부서코드pk, 부서명)


4. 보이스코드 정규화
- 결정자 모두가 후보키

5. 4차 정규화
- 다치 종속 제거

6. 5차 정규화
- 조인 종속 제거

*시험, 실무에서는 3차까지만 이해&실습 가능하면 충분함. 이후는 흐름과 키워드만 암기.*


## 반정규화

**반정규화** : 정규화를 통해 분리된 엔터티를 다시 합치거나 중복을 허용하는 작업

- 물리적 모델링에서 수행된다.
- 조회 성능 향상을 목적으로 함.(정규화 하면 엔터티가 분리됨. 데이터 조회 시 여러 엔터티를 봐야 해서 성능 떨어질 수도.)


###  반정규화 결정 단계

선택 사항이며 가급적 피해야 한다.

1. 정규화 수행 : 논리 모델링 단계. 우선 정규화 필수.
2. 조회 성능 분석 : 물리 모델링 단계. *조회가 느리다고 판단되는* 구간을 분석
3. 대안 검토 : 인덱스, 파티셔닝 등 다른 방안을 우선 검토
4. 최종 반정규화 적용 : 반정규화 시 중복과 이상현상을 개발자가 직접 관리해야 함.

### 성능

| 구분 | 정규화 | 반정규화 |
|------|------|------|
| 조회 성능 | 향상 or 감소 | 향상 가능 |
| 조작 성능 | **향상** | **감소** |
| 데이터 품질 | 중복x. 무결성 유지 | 중복 o. 품질 저하 우려 |
| 사용 시점 | 논리 모델링 단계(필수) | 물리 모델링 단계(선택) |

*조작 성능 = 삽입/수정/삭제 성능*
