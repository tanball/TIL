# 관계형 데이터베이스

## 용어

1. **데이터베이스** : 구조화된 형태로 데이터를 저장해 두는 집합.
2. **DBMS** : DataBase Management System. 기존 데이터베이스에 관리 기능을 추가한 소프트웨어.
- 데이터를 저장, 조회하는 기능을 제공하는 데이터베이스
- 백업, 복구, 보안, 동시성 제어 등 다양한 기능 제공
- 오라클, mySQL, SQL server 등(SQLD는 대개 Oracle 기반)
3. **RDBMS** : Relational DBMS. 관계형 데이터베이스. 엔터티 간의 관계를 기반으로 데이터의 논리적인 의미를 처리하는 DMBS.
- 그 외에는 계층형 DBMS, 네트워크형 DBMS 등이 존재.
4. **테이블** : RDBMS에서 데이터를 저장하고 조회하는 2차원 배열 형태의 저장소. 모델링 단계에 따라 이름이 다름
- 개념적 모델 : Entity | 논리적 모델 : Relation | 물리적 모델 : Table
5. **SQL** : Structured Query Language. 직역 시 '구조화된 질의 언어'. 데이터베이스에 특정 구조(형식)에 맙게 질문하는 언어. 

## SQL 문법 종류

| 분류 | 명칭 | 명칭 | 설명 | 예시 |
|------|------|------|------|------|
| DDL | Data Definition Language | 데이터 정의어 | 데이터 구조 정의 | CREATE, ALTER, DROP, RENAME, TRUNCATE |
| DML | Data Manipulation Language | 데이터 조작어 | 데이터 조작 | SELECT, INSERT, UPDATE, DELETE, MERGE |
| DCL | Data Control Language | 데이터 제어어 | 권한 부여 및 회수 | GRANT, REVOKE |
| TCL | Transaction Control Language | 트랜잭션 제어 언어 | 트랜잭션 제어 | COMMIT, ROLLBACK, SAVEPOINT |

*빈출*

## Standard SQL

SQL은 여러 DBMS간 호환을 위해 표준 규격을 제공.

ISO(국제표준화기구)와 ANSI(미국국립표준협회)가 만들었다.

단, 강제 사항은 아니기에 DBMS별로 차별화한 문법은 별도로 학습해야.

수학의 집합 이론과 관계 이론을 근간으로 함.

### 일반 집합 연산자

1. UNION : Union(합집합). 두 테이블의 중복을 제거한 전체 데이터 집합
2. INTERSECT : Intersection(교집합). 두 테이블에 모두 존재하는 데이터만 추출
3. MINUS(or EXCEPT) : Difference(차집합). 한쪽 테이블에는 존재하지만 다른 쪽에는 없는 데이터 추출
4. CROSS JOIN : Product(곱집합). 모든 조합의 데이터를 생성(cartesian product)

*MINUS는 오라클, EXCEPT는 SQL Server등에서 사용.*

### 순수 관계 연산자
1. WHERE : select(ion). 테이블에서 필요한 컬럼(열)만 추출. 시그마 기호.
2. SELECT : project(ion). 테이블에서 필요한 튜플(행)만 추출. 파이 기호.
3. JOIN : 두 테이블을 조건에 따라 연결. 보우타이 기호.
4. X : divide. 집합 나눗셈. 현재 sql에서는 사용하지 않음.
