# GROUP BY, HAVING절

## GROUP BY
(where로 필터링 된) 데이터들을 특정 컬럼을 기준으로 그룹화.

> 예시
> 
> ```
> SELECT 소속반, COUNT(*) AS 반별인원수
> FROM 수강생정보
> GROUP BY 소속반;
>```

### 주의사항
GROUP BY 사용 시, SELECT 절에는 group by에 명시한 컬럼, 표현식, 집계 함수만 사용 가능.

* '소속반'컬럼으로 그룹화했다면, '소속반'외의 다른 컬럼(학생이름 등)은 조회 불가
* 그룹화 할 때, 컬럼이 아니라 그 컬럼을 가공한 표현식을 기준으로 그룹화할 수도 있다. 이 때 그 표현식을 select절에서 사용할 수 있다.(ex group by substr(학생이름, 1, 1) : 학생 성 기준으로 그룹화)
* COUNT()로 대표되는 **집계함수**는 여러 행을 입력 받아 하나의 결과를 반환한다.(다중행 함수)

### 집계함수
다중행 함수(집계 함수, 그룹 함수, 윈도우 함수 등)의 일종

종류
- COUNT(*) : 전체 행 수. null포함.
- COUNT(컬럼) : 해당 컬럼에 있는 null을 제외한 수
- SUM(컬럼) : 합계. null제외. 숫자에 사용 가능
- AVG(컬럼) : 평균. null제외. 숫자에 사용 가능
- MAX(컬럼) : 최대값. null제외. 숫자/문자/날짜에 사용 가능
- MIN(컬럼) : 최소값. null제외. 숫자/문자/날짜에 사용 가능


포인트
1. group by 없어도 select 절에 집계 함수를 사용할 수 있다.(테이블 전체를 하나의 그룹으로 보고 수행.) 단, 일반 컬럼과 병용하려면 그룹화 필수.
2. 집계함수가 공집합(조건에 맞는 행 없음)을 처리할 때, 대부분 NULL을 반환하나 COUNT는 0을 반환.
3. 집계함수 입력값으로 [DISTINCT](SELECT문.md#distinct)를 사용할 수 있다. 중복 제거 먼저->집계 처리

## HAVING
group by로 그룹화된 결과에 대해 조건 필터링을 할 수 있게 도와주는 구문

where과 비슷함. 그러나, where에서는 집계 함수 조건을 쓸 수 없어서 having이 필요

### 주의사항
group by에 사용된 컬럼 또는 집계 함수 결과만 사용할 수 있다.
